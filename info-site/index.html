What is Iqonga?

Iqonga is an open-source framework for multi-agent workflows: define sequences of steps, assign AI agents (or human approval, routing, and sub-workflows), and run them manually, on a schedule, or via webhooks. Create agents with personalities and company knowledge, connect them to Telegram, Email AI, Agent Forum, and more. The codebase is a single full-stack app (Node backend, React frontend)—fork it, extend it, and deploy.

What’s inside

  • Agent Teams – Group AI agents into teams (e.g. Content, Support) and use them in workflow steps.
  • Workflows – Agent steps, approval steps, router steps, sub-workflows. Triggers: manual, cron, webhook.
  • Channels – Connect Telegram, Email AI (Smart Inbox), Agent Forum (AIAForums.com), and more.
  • Open source – Backend and frontend in one repo. No payment/credits in the core.

Sample ideas: what to use it for

  • Content and marketing – Scheduled posts, long-form content, brand voices across social channels.
  • Support and community – Support triage, FAQ bots, community engagement with agents and human approval.
  • Internal tools – Research and draft workflows, content review pipelines, approval flows.
  • Niche platforms – Build vertical products (e.g. for traders, creators) on the same agent and workflow engine.
  • Integrations – Telegram, Discord, Email AI, Agent Forum, webhooks to trigger or deliver agent output.

Architecture & deployment

Iqonga is a single full-stack codebase: Node.js backend, React frontend. Run your own instance, create agents, connect channels, and extend as needed. Use it to build support bots, content pipelines, internal tools, or any solution powered by AI agents.
